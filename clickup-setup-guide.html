ClickUp Container Tasks Setup Guide for SOAHC-NIKCUF-Unity

This guide will help you set up container tasks in ClickUp for organizing your Unity project commits.

Setup Steps

  1. Log in to your ClickUp account
  2. Navigate to your workspace
  3. Create a new List called "Development Categories"
  4. Create the container tasks as described below
  5. Pin these tasks to the top of your workspace for easy access

Container Tasks to Create

Bug Fixes

Suggested ID: TASK-101

Description: Container for all bug fix commits. Use this task ID when committing code that fixes bugs, resolves issues, or addresses unexpected behavior.

Example commit:

git commit -m "Fix player collision detection #TASK-101"

Features

Suggested ID: TASK-102

Description: Container for all new feature implementations. Use this task ID when committing code that adds new functionality, systems, or capabilities to the game.

Example commit:

git commit -m "Add new weapon system #TASK-102"

Refactors

Suggested ID: TASK-103

Description: Container for code refactoring and improvements. Use this task ID when committing code that improves structure, readability, or maintainability without changing functionality.

Example commit:

git commit -m "Refactor enemy AI for better maintainability #TASK-103"

Performance Optimizations

Suggested ID: TASK-104

Description: Container for performance-related changes. Use this task ID when committing code that improves game performance, reduces memory usage, or optimizes resource loading.

Example commit:

git commit -m "Optimize rendering pipeline for better FPS #TASK-104"

UI/UX Improvements

Suggested ID: TASK-105

Description: Container for user interface and experience changes. Use this task ID when committing code that enhances the game's UI, menus, HUD, or overall user experience.

Example commit:

git commit -m "Update main menu layout and animations #TASK-105"

Art & Assets

Suggested ID: TASK-106

Description: Container for art-related commits. Use this task ID when committing new or updated art assets, models, textures, or visual effects.

Example commit:

git commit -m "Add new character models and textures #TASK-106"

Audio

Suggested ID: TASK-107

Description: Container for audio-related commits. Use this task ID when committing new or updated sound effects, music, or audio systems.

Example commit:

git commit -m "Add new background music for forest level #TASK-107"

Documentation

Suggested ID: TASK-108

Description: Container for documentation updates. Use this task ID when committing changes to documentation, comments, or README files.

Example commit:

git commit -m "Update installation instructions in README #TASK-108"

Note: The actual task IDs in ClickUp will be automatically generated and will look different from the suggested IDs above. Once you create the tasks, make note of their actual IDs for use in commit messages.

Additional Tips

  • Use ClickUp's "Pin" feature to keep these container tasks easily accessible
  • Consider using different colors for each category to make them visually distinct
  • You can add checklists within each container task to track specific items
  • Use the description field to provide more detailed guidelines for each category
  • Consider setting up ClickUp Automations to update task statuses based on GitHub activity

Remember: The GitHub integration will automatically link your commits to these tasks when you include the task ID in your commit messages with the # symbol.
